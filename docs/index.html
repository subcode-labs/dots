dots

Dotfiles that just work

Minimal dotfile management

Store your real dotfiles in a clean repo, keep your machine in sync, and avoid template-heavy workflows.

Key features

Edit-in-place

Your dotfiles stay where they belong while being tracked in one repo.

Readable repos

Keep files with their original names and avoid template syntax noise.

Simple YAML

One manifest at ~/.dots/dots.yaml maps sources to targets.

Quick install

Install via Go in a single command:

go install github.com/subcode-labs/dots@latest
